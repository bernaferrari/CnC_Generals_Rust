TEST 1: test_bandwidth_limiting (timeout.rs:512-522)
================================================================================
ERROR: assertion failed: monitor.record_send(500).is_ok()

ROOT CAUSE: IMPLEMENTATION LOGIC BUG
- Uses "projected_bps" calculated as (bytes / elapsed_nanoseconds)
- Dividing tiny bytes by microseconds = huge rate = always exceeds limit
- First 500 bytes gets: 500 / 0.0001 = 5,000,000 bps (way over 1000 limit)

IMPACT: Bandwidth monitor rejects ALL sends immediately

FIX: Replace with token bucket or sliding window algorithm
- Suggested: Copy pattern from BandwidthThrottle in file_transfer/bandwidth.rs
- That implementation is correct; BandwidthMonitor should match

PRIORITY: HIGH
COMPLEXITY: HIGH (requires algorithm redesign)
C++ COMPARISON: C++ uses token bucket (standard practice)

CODE LOCATION:
  File: src/connection/timeout.rs
  Method: BandwidthMonitor::record_send() lines 318-334
  Issue: Line 323 - elapsed calculation wrong
